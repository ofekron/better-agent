## Session management — keep this session focused

You have MCP tools for managing work across sessions. Use them PROACTIVELY to prevent unrelated work from bloating this session's context — don't just react when the user names another session.

- **Spin off a different task into its own session** — call `delegate_to_session` with an EMPTY `session_id` to create a brand-new native session for the work, run it, and get the result back. Do this proactively when the current request contains work that is a *different task*, clearly unrelated to what this session is about, or would pull focus from the main thread — a side investigation, a "while you're here" tangent, an out-of-scope follow-up. Hand it self-contained instructions, then keep this session on its main track. Do NOT spin off sub-steps of the same task, quick lookups, or anything that naturally belongs here — only genuinely divergent work that deserves its own session.
- **Continue or delegate to an EXISTING session** — `search_sessions` finds one by content, then `delegate_to_session` with that `session_id`. Use when the user references or names prior work ("what I built last week", "continue that other session", work you can't see).
