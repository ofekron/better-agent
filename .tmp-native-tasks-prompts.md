# Native Session Tasks Feature - Prompts and Instructions Search Results

## Summary
Native sessions in Better Agent receive task-related instructions through TWO mechanisms:
1. **Runtime injection** via `_append_todo_reminder()` (TODO reminder block)
2. **Project rules** in CLAUDE.md (TaskCreate/TaskUpdate mandate)

Unlike manager/team orchestration modes, native mode does **NOT** have a bootstrap system prompt file.

## 1. TODO Reminder Injection (Runtime Prompt Addition)

### Location: `/Users/ofekron/better-claude/backend/turn_helpers.py`

**Lines 147-201** - Core TODO reminder implementation:

```python
_UNFINISHED_TODOS_OPEN = "<bc-todo-reminder>"
_UNFINISHED_TODOS_CLOSE = "</bc-todo-reminder>"

def _append_todo_reminder(cli_prompt: str, session: dict) -> str:
    """Append tagged session todo state to `cli_prompt`.

    Caller-side gates (user_initiated, empty-prompt) decide whether
    to call this. Empty and all-completed lists leave the prompt unchanged.
    """
    todos = _session_work_items(session)
    unfinished = [
        todo for todo in todos
        if todo.get("status") != "completed"
    ]
    if not unfinished:
        return cli_prompt

    lines = []
    for todo in todos:
        status = html.escape(
            " ".join(str(todo.get("status") or "pending").split())
        )
        content = html.escape(
            " ".join(str(todo.get("content") or "Untitled todo").split())
        )
        lines.append(f"- [{status}] {content}")
    unfinished_block = (
        f"{_UNFINISHED_TODOS_OPEN}\n"
        "Current todo state from this session:\n"
        + "\n".join(lines)
        + f"\n{_UNFINISHED_TODOS_CLOSE}"
    )

    return cli_prompt.rstrip() + "\n\n" + unfinished_block
```

**What it does:**
- Reads `session.current_todos` and `session.current_tasks` fields
- Formats incomplete items as a list with HTML-escaped content
- Wraps in `<bc-todo-reminder>...</bc-todo-reminder>` tags
- Appends to the CLI prompt sent to the model

## 2. Native Orchestration Strategy (Prompt Wrapper)

### Location: `/Users/ofekron/better-claude/backend/orchs/native/__init__.py`

**Lines 38-56** - `wrap_cli_prompt()` method:

```python
def wrap_cli_prompt(self, *, cwd: str, prompt: str, session: dict) -> str:
    if (session.get("orchestration_mode") or "team") not in ("team", "manager"):
        return prompt
    # Bare (TestApe-isolated) sessions get an EMPTY system prompt — the
    # caller's prompt is the complete contract, no manager bootstrap.
    if session.get("bare_config"):
        return prompt
    from orchs.manager import bootstrap as manager_bootstrap
    is_first_turn = session.get("agent_session_id") is None
    return manager_bootstrap.build_wrapped_prompt(
        cwd,
        prompt,
        is_first_turn,
        self_session_id=str(session.get("id") or ""),
        self_role="manager",
        self_description=str(session.get("name") or "manager"),
        manager_session_id=str(session.get("id") or ""),
        manager_description=str(session.get("name") or "manager"),
    )
```

**Key insight:**
- Native mode returns `prompt` unchanged (identity function)
- Only manager/team modes get bootstrap prompts injected
- Line 190 comment mentions "open-todo reminder" as a user-facing nudge

## Key Findings Summary

### What Native Sessions Receive About Tasks:

1. **<bc-todo-reminder> block** injected at runtime showing:
   - Current TODO state from session
   - List of incomplete work items with status
   - HTML-escaped content

2. **CLAUDE.md project rules** (if loaded as context) stating:
   - TaskCreate/TaskUpdate are MANDATORY
   - All work must be tracked
   - No task is too small

3. **Session bridge instructions** (via capability context) teaching:
   - How to isolate different tasks into separate sessions
   - When to delegate vs. handle inline

### What Native Sessions Do NOT Receive:

- **No bootstrap system prompt** (unlike manager/team modes)
- **No explicit TaskCreate/TaskUpdate tool descriptions** in system prompts
- The tasks feature is documented implicitly through project rules, not explicitly through tool descriptions

### Architecture Notes:

- Native mode is intentionally "plain" - no bootstrap injection
- Task tracking is enforced through project conventions (CLAUDE.md)
- TODO reminder is runtime context, not system prompt
- Manager mode gets explicit bootstrap; native mode does not
