01 Executive summary

The dominant signal this cycle is provider churn at the integration boundary — the seams Better Agent depends on moved underneath it, and two demand immediate action.

  • Google retired Gemini CLI on Jun 18 and replaced it with Antigravity CLI. Better Agent ships a Gemini provider, so its integration target has been renamed and partly re-architected (server-side harness, "Antigravity plugins"). This is a live parity break, not a nice-to-have.
  • The Claude Messages API now accepts system entries inside the messages array (Jul 15 platform note, lit up on Opus 4.8 / Fable 5 / Mythos 5) — i.e. mid-conversation system messages. This is the native primitive for Better Agent's "per-turn dynamic tool/system-prompt mechanism," letting instructions be updated mid-task without breaking context.
  • Claude Code gained Artifacts: a session publishes its work to a live, interactive, versioned web page at a private URL that updates in place. Directly relevant to Better Agent's routine-output publishing and bcfile artifact surface — the same idea, productized.
  • MCP security hardened into a real standard: the OWASP MCP Top 10 is published (MCP03 = tool poisoning), 82% of tested MCP servers were vulnerable to path traversal, and NSA MCP guidance now maps cleanly to OWASP. Better Agent's ambient-MCP + tool-execution surface has a concrete conformance checklist.
  • The 2026-07-28 MCP spec is 11 days out — final GA lands during the next reporting cycle; the DRAFT→final migration window closes now.

Highest-value bets this cycle: (1) re-point the Gemini provider at Antigravity CLI before the legacy path rots further; (2) wire mid-conversation system messages into the per-turn dynamic mechanism across Claude/Codex/Gemini; (3) run the ambient-MCP surface through the OWASP MCP Top 10 (path-traversal + tool-poisoning gates) as a security pass.

Rank 1 · Provider-parity alert

Gemini CLI retired → Antigravity CLI

Google deprecated Gemini CLI on Jun 18, 2026, replacing it with Antigravity CLI — a re-architected surface with a server-side harness and "Antigravity plugins" (formerly Skills/Hooks/Subagents/Extensions).

91
Integration shape
Re-point Better Agent's Gemini provider at the Antigravity CLI binary/harness; reconcile the plugin model with the existing Gemini runner. Preserve the Claude-shaped event normalization already in runner_gemini.py.
Why now
Better Agent explicitly ships Gemini parity; the legacy CLI is retired and the integration target has moved. Every day this drifts, the Gemini provider risks rotting against a dead binary.
Risks
Provider parity Antigravity is new — pin to a versioned binary, detect capability shape, and fail closed on drift. Keep Claude/Codex parity intact while migrating; never silently degrade the Gemini path.

Rank 2 · Provider capability

Mid-conversation system messages (Messages API)

Per the Jul 15 platform note, the Messages API now accepts system entries inside the messages array (live on Opus 4.8 / Fable 5 / Mythos 5) — instructions can be updated mid-task without breaking context.

88
Integration shape
This is the native primitive for Better Agent's "per-turn dynamic tool/system-prompt mechanism." Inject refreshed instructions or tool availability mid-turn across Claude/Codex/Gemini instead of rebuilding the system block, preserving session continuity.
Why now
Previously the only way to change instructions mid-task was a full system-prompt rewrite or a resumed session; this gives a first-class insertion point — exactly the seam Better Agent's harness already wants to use.
Risks
Prompt injection Mid-conversation system text is trusted context — never blend untrusted/tool-returned content into it. Treat only harness-authored text as eligible; keep provider parity (Codex/Gemini may need an adapter until they ship an equivalent).
Evidence
Introducing Claude Opus 4.8 · Claude platform release notes (Jul 15)
Rank 3 · Security

OWASP MCP Top 10 — 82% path-traversal exposure

The OWASP MCP Top 10 is published (MCP03 = tool poisoning), and a 2026 audit found 82% of tested MCP servers vulnerable to path traversal. A concrete conformance checklist now exists.

85
Integration shape
Run Better Agent's ambient-MCP preset and tool-execution surface through the OWASP MCP Top 10: enforce path-confinement (paths.bc_home(), reject ../abs/symlink escapes), validate tool descriptions for poisoning, and add a vetting gate for community MCP servers.
Why now
The 82% number shows path traversal is the de-facto default failure; Better Agent's security-first stance (paths.bc_home() confinement already exists) makes this a focused hardening pass, not a redesign.
Risks
Tool poisoning hides attacker instructions in tool descriptions invisible to users but readable by the model — every ambient-MCP preset is an injection vector. Fail closed on untrusted config.
Evidence
OWASP MCP Top 10 · MCP security statistics 2026 · Cycode: OWASP MCP Top 10 2026
Rank 4 · Artifact surface

Claude Code Artifacts — live shareable session pages

Claude Code now publishes session work to a live, interactive, versioned web page at a private URL that updates in place (beta, Team/Enterprise). Use cases: PR walkthroughs, living dashboards.

78
Integration shape
Extend Better Agent's existing routine-output publishing + bcfile artifact surface into a "share session as a live page" view — a read-only, updating projection of a session/turn, behind a private shareable link. Backend remains source of truth; the page is a projection.
Why now
Both platforms converging on "session → live web page" validates the artifact surface Better Agent already half-builds for routine reports; productizing it closes a visible gap.
Risks
Data egress A shared page can leak session content/secrets. Default private, opt-in sharing, scrub secrets/PII before publish, short-lived signed links, and never expose raw filesystem paths.
Evidence
VentureBeat: Artifacts update · Claude Help: Artifacts

06 Risks & security cross-cut

Provider drift is the live risk this cycle

Two of today's top items moved the integration boundary.

  • Antigravity CLI (Rank 1): the Gemini provider target was renamed/re-architected; pin a versioned binary, detect capability shape, fail closed on drift, and keep Claude/Codex parity intact during migration.
  • Mid-conversation system messages (Rank 2): mid-turn system text is trusted context — never blend tool-returned/untrusted content into it; harness-authored text only. Codex/Gemini need adapters until they offer an equivalent.

MCP attack surface — the 82% problem

Ambient-MCP presets and tool descriptions are the new injection boundary.

  • Path traversal (Rank 3): confine all FS access through paths.bc_home(); reject .., absolute escapes, symlink escapes; the 82% rate means assume-hostile.
  • Tool poisoning (MCP03): attacker instructions hide in tool descriptions invisible to users but readable by the model — vet every community MCP server; fail closed on untrusted config.
  • Shareable artifacts (Rank 4): a live session page is an egress path — default private, scrub secrets/PII, signed short-lived links, no raw paths.
