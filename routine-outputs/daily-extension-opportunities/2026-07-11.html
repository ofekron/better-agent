Executive summary TL;DR

The single highest-value opening this week sits at the intersection of MCP's explosive growth and its acute security gap. The official MCP Registry neared 9,652 server records (May 2026) while independent analysis found 36.7% of ~7,000 MCP servers vulnerable to SSRF, with documented STDIO-RCE and supply-chain compromises (postmark-mcp, ClawHub). Better Agent — which already spawns real CLI subprocesses and treats security as the overriding constraint — is uniquely positioned to ship a vetted MCP registry browser + hardened gateway as a first-class extension.

Three further opportunities are high-feasibility same-turn wins: a spec-driven-development runtime skill (GitHub Spec Kit / Agent OS v2 patterns), native parallel-worktree orchestration (aligning with Claude Code's --worktree/--tmux and the community parallel-worktrees skill), and wiring Opus 4.8 dynamic workflows + /goal as a provider capability.

One item is a must-handle risk, not optional: Google retired the open-source Gemini CLI in June 2026 for a closed-source successor. Better Agent is multi-provider (Claude/Codex/Gemini), so Gemini parity needs a deliberate migration path — the only provider of the three whose open CLI disappeared.

Top opportunities — detail cards

#1 · MCP Registry browser + security vetting Extension / MCP

The official MCP Registry (registry.modelcontextprotocol.io) is now the centralized, Anthropic-backed catalog nearing 10K servers. Better Agent could add a first-class registry browser: search, inspect, one-click install — but with a vetting gate (sig/publisher checks, egress allow-list, scoped tokens) before any server is attached to a session.

Integration shapeNew mcp_registry extension: REST browse → install dialog → scoped attach to a session/worker. Reuse provider_bridge MCP plumbing already in the backend.
ComplexityMedium — registry API is public; the vetting policy + UI is the bulk of the work.
Why nowRegistry crossed 9.6K records (May 2026); GitHub MCP Registry (Sep 2025) made discovery trivial — demand is here.
RisksTool poisoning, over-privileged servers, SSRF (36.7% of sampled servers). Mitigate with sandbox + pinning (see #4).
Sources: Official MCP Registry · MCP Registry about · MCP Adoption Statistics 2026 (9,652 records) · GitHub MCP Registry announcement · BlueRock: 36.7% SSRF

#2 · Spec-driven-development runtime skill Runtime skill

Spec-driven development (SDD) has matured into a documented methodology: GitHub's open-source Spec Kit, Microsoft's spec-first tooling, and Agent OS v2 (tool-agnostic, works with Claude Code/Cursor). Better Agent already has a requirement-corpus pipeline — an SDD skill that turns a requirement into spec → plan → tasks → deliverable is a natural, high-feasibility fit.

Integration shapeRuntime skill (spec-driven-dev) + workflow: /spec command produces a spec doc, then delegates plan/tasks to existing workers. Pairs with the requirements MCP.
ComplexityLow–Medium — mostly prompt + workflow glue over existing worker delegation.
Why nowGitHub + Microsoft both shipped open SDD toolkits in late 2025; 2026 saw 30+ agentic-coding frameworks converge on it.
RisksSpec drift from code; mitigate by treating spec as a disposable projection of the requirement corpus (single source of truth).
Sources: GitHub Blog — Spec Kit · Microsoft Developer — spec-first · Augment — What is SDD · arXiv: SDD executable contracts (2026)

#3 · Native parallel-worktree orchestration Workflow / skill

Claude Code's --worktree/--tmux flags and the community parallel-worktrees skill show the winning isolation pattern: one .git DB, N working directories, N agents. Reports of 200–1,000 parallel subagents across dozens of repos. Better Agent already does multi-agent/worktree delegation — formalize it as a documented, first-class workflow.

Integration shapeBuilt-in parallel-worktrees skill + manager workflow: fan out N workers across git worktrees, gate merges via git_ops lock, aggregate results. Reuses existing EnterWorktree + worker delegation.
ComplexityMedium — orchestration exists; the gap is a polished user-facing workflow + conflict policy.
Why nowMainstream in 2026 with documented scale (200+ subagents / 35 repos).
RisksBranch conflicts, lock contention, runaway cost. Gate with bounded concurrency caps + precise file_edit: locks.
Sources: parallel-worktrees skill (GitHub) · Developers Digest — 200+ subagents across 35 repos (May 2026) · Tembo — Claude Code multi-agent 2026 · Towards AI — 1,000 subagents

#4 · Hardened MCP gateway (sandbox + supply-chain) Security / MCP

CSA Labs documented STDIO-based RCE in MCP (April 2026); Bishop Fox detailed supply-chain compromises of postmark-mcp and ClawHub; CoSAI cataloged "MCP-T11: Supply Chain & Lifecycle Security Failures." Better Agent's CLAUDE.md already mandates fail-closed security for subprocess spawning — a hardened MCP gateway is the defensive complement to #1.

Integration shapeMCP gateway layer: version-pinned installs, egress allow-list, per-server capability scoping, stdio sandbox (no shell, arg-list only), provenance/signature checks before attach.
ComplexityHigh — real sandboxing (OS-level) is non-trivial; start with policy + pinning + egress.
Why nowActive exploitation documented in 2026; CoSAI + CSA published mitigations this year.
RisksOver-blocking breaks legit servers; mitigate with per-session opt-in scopes and audit logs.
Sources: CSA Labs — STDIO RCE & supply chain (Apr 2026) · Bishop Fox — supply-chain risks · CoSAI — MCP Security (MCP-T11) · CSA — 7 MCP risks · Pomerium — MCP security 2026

#5 · Gemini CLI retirement → successor migration Provider parity (risk)

Google retired the open-source Gemini CLI for most public users in June 2026 (105K+ stars) in favor of a closed-source successor. Better Agent is multi-provider (Claude/Codex/Gemini) and ingests Gemini via its CLI stream. This is the only provider of the three whose open CLI vanished — a parity decision, not a feature choice.

Integration shapeMigrate provider_gemini + runner_gemini to the closed-source successor's interface, or adopt the Gemini API directly. Preserve the Claude-shaped event normalization funnel.
ComplexityHigh — depends on successor's stability/API surface; may need a fork-in resilience plan.
Why nowRetirement was June 2026; the longer Gemini parity drifts, the more the convergence invariant is at risk.
RisksClosed-source = less control, possible telemetry/licensing constraints. If parity impossible, stop and ask per project rules.
Sources: google-gemini/gemini-cli (GitHub) · Gemini CLI changelogs (v0.37, Apr 2026) · DEV — Gemini CLI shutdown (June 2026) · State of CLI coding agents mid-2026

#6 · Universal agent-memory layer (mem0 / cognee) Product improvement

mem0 is the leading open-source universal memory layer for agents; cognee provides a persistent memory framework. Better Agent already has a requirement corpus + per-session state — a memory layer could unify cross-session recall (decisions, preferences, facts) behind one owner, consistent with the single-source-of-truth rule.

Integration shapeAdopt mem0 as an optional memory backend; project cross-session facts as a disposable cache over the requirement corpus (source wins on conflict).
ComplexityMedium — integration is straightforward; the design discipline is keeping one owner.
Why nowmem0's 2026 state-of-memory benchmark matured the category; cognee published persistent-memory guides.
RisksSecond source of truth drift; mitigate by never persisting derived state, only facts, and invalidating on conflict.
Sources: mem0ai/mem0 (GitHub) · mem0 — State of AI Agent Memory 2026 · Cognee — persistent memory · ML Mastery — 6 memory frameworks 2026

Risks & security considerations cross-cutting

MCP supply chain — Compromised servers (postmark-mcp, ClawHub) and registry tool-poisoning. Pin versions, verify publisher, fail closed on unsigned servers.
STDIO RCE — CSA Labs (Apr 2026) documented stdio-based remote code execution via MCP. Never shell-concatenate untrusted args; pass arg lists; confine to bc_home().
SSRF exposure — 36.7% of ~7,000 MCP servers SSRF-vulnerable (BlueRock). Egress allow-lists and network isolation for any attached server.
Over-privileged agents — CSA + TrueFoundry flag tool abuse & cross-agent contamination. Least-privilege per-server scopes; per-session opt-in.
Provider-parity drift — Gemini CLI retirement (June 2026) breaks the open-CLI assumption. Decide migration path before parity silently degrades.
Parallel-agent cost/conflict — 200–1,000 subagent runs risk runaway cost and branch conflicts. Bounded concurrency caps + git_ops + file_edit: locks.

These map directly to Better Agent's existing security rules (CLAUDE.md: security is the overriding constraint, fail closed, paths.bc_home() confinement, no secret leakage). Opportunities #1 and #4 are the offensive and defensive halves of the same MCP story.
