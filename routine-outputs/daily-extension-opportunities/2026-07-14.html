01 Executive summary

This week's signal is dominated by two foundational shifts that map directly onto Better Agent's architecture. First, the MCP 2026-07-28 spec release candidate makes the protocol stateless (no Mcp-Session-Id, no initialize handshake), promotes Extensions to first-class, and revises agent authentication — all of which align with Better Agent's existing per-extension token model and ambient-MCP proxying. Second, Anthropic's Managed Agents primitives — scheduler, dreaming, and rubric-graded outcomes — are a near-perfect fit for Better Agent's routines + requirements substrate, with a reported +8.4% task improvement from the outcomes loop alone.

On the security axis, MCP tool-poisoning / indirect prompt-injection (OWASP LLM01) is now the most-cited client-side vulnerability in academic and industry analyses — directly relevant given Better Agent spawns subprocesses and surfaces untrusted tool descriptions. The official MCP Registry now offers a canonical REST API with verified namespaces, and the Agent Skills (SKILL.md) ecosystem has matured into multiple 1,000+ skill marketplaces.

Highest-value bet this cycle: adopt the MCP 2026-07-28 stateless + extensions model in the MCP bridge, and stand up a rubric-graded "outcomes" loop over routine runs. Both are foundational, reversible-in-part, and compound with existing infrastructure.

03 Top opportunity cards

Rank 1 · Spec / foundational

MCP 2026-07-28: stateless protocol + first-class Extensions

The largest MCP revision since launch. SEP-2567 removes Mcp-Session-Id and the initialize handshake; "Extensions" become a core concept; auth is revised for stateless, multi-instance servers.

95
Integration shape
Update the MCP bridge/proxy to drop session-id lifecycle, add Extensions negotiation, and adopt the revised agent-auth scope model. Maps cleanly onto Better Agent's per-extension token identity.
Why now
Better Agent already routes identity via per-extension tokens and proxies many MCP servers — the new auth model is a natural fit and the stateless core unlocks horizontal scaling of its MCP fan-out.
Risks
Breaking SEP-2567 is a hard break for any session-id-based lifecycle; needs a migration path and dual support during RC. Provider parity across Claude/Codex/Gemini must be preserved.
Evidence
MCP blog · release · spec author summary · WorkOS breakdown
Rank 2 · Orchestration

Managed Agents: dreaming + rubric-graded outcomes

Anthropic shipped three primitives for managed agents: a scheduler, a dreaming pass (consolidates lessons across sessions/memory), and outcomes (rubric-graded self-correction, +8.4% cited).

86
Integration shape
Extend the routines assessment field into a rubric + independent evaluator loop ("outcomes"); add a "dreaming" runtime skill that periodically distills session insights into the requirements corpus.
Why now
Better Agent already has routines (scheduler), the requirements pipeline, and per-session memory — dreaming/outcomes are direct extensions of existing surfaces, not new subsystems.
Risks
Extra evaluator passes cost tokens; evaluator quality gates the loop; must keep cross-provider parity and fail-closed behavior.
Evidence
Anthropic engineering · The New Stack · Developers Digest
Rank 3 · Security

MCP tool-poisoning & indirect prompt-injection defenses

Malicious instructions hidden in MCP tool metadata (invisible to users, visible to the model) are now OWASP LLM01 and the most-cited client-side MCP vulnerability in 2026 literature.

84
Integration shape
Add a tool-description inspection/sanitization gate before MCP registration, an allowlist mode, runtime anomaly monitoring, and route sensitive tool calls through Better Agent's existing approvals surface.
Why now
Better Agent spawns real subprocesses and renders untrusted tool descriptions — defense-in-depth here is a direct project priority ("security is the top priority").
Risks
Over-aggressive sanitization can break legitimate tools; needs allowlist + human-in-the-loop tuning to avoid false positives.
Evidence
Invariant Labs · OWASP LLM01 · Elastic Security Labs
Rank 4 · Discovery

Official MCP Registry as canonical discovery

The official registry now ships a canonical REST API with verified namespaces, positioned as the "app store for MCP servers."

72
Integration shape
A registry-backed browser with one-click install into Better Agent's ambient-MCP surface, preferring verified-namespace servers; reuse provider-config-sync plumbing.
Risks
Registry API is still evolving; namespace-trust model must be reflected in the UI (verified vs. community).
Evidence
Official registry · registry repo · GitHub changelog
