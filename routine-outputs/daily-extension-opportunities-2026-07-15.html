01 Executive summary

This cycle's signal is dominated by a foundational protocol shift and a maturing security consensus, both of which map directly onto Better Agent's architecture. The MCP 2026-07-28 spec release candidate introduces a stateless core (dropping Mcp-Session-Id and the initialize handshake) and promotes Extensions to a first-class concept — a near-perfect fit for Better Agent's existing per-extension token identity and ambient-MCP proxying. Meanwhile the official MCP Registry is nearing ~9,650 server records and has become the canonical discovery index.

On the security axis, the OWASP MCP Top 10 is now published with MCP03 Tool Poisoning as a core entry, reinforced by a July 2026 CSA research note and NSA-guidance mappings. Given Better Agent spawns real CLI subprocesses and renders untrusted tool descriptions, defense-in-depth here is a direct project priority — and pairs naturally with open-source code-execution sandboxes (E2B, Daytona) that can isolate the destructive commands the harness already issues.

The capability frontier is expanding too: Anthropic's Managed Agents primitives (scheduler / dreaming / rubric outcomes) extend what Better Agent's routines + requirements already do; the Agent Skills (SKILL.md) ecosystem has consolidated around real catalogs (anthropics/skills, SkillsMP, ClaudeSkills.info); and open-source agent eval frameworks (DeepEval, promptfoo, agentevals) offer a ready-made assessment layer.

Highest-value bets this cycle: (1) adopt the MCP 2026-07-28 stateless + Extensions model, (2) stand up a tool-poisoning inspection gate plus optional sandboxed execution, and (3) wire an open eval framework into routine assessment. All three are foundational, reversible-in-part, and compound with existing surfaces.

04 Top opportunity cards

Rank 1 · Spec / foundational

MCP 2026-07-28: stateless protocol + first-class Extensions

The next MCP spec release candidate makes the protocol stateless (no Mcp-Session-Id, no initialize handshake) and promotes Extensions to a core concept.

95
Integration shape
Update the MCP bridge/proxy to drop session-id lifecycle, add Extensions negotiation, and route identity via the existing per-extension token model.
Why now
RC lands 2026-07-28; the registry is already ~9,650 servers and growing. Stateless core unlocks horizontal scaling of Better Agent's MCP fan-out.
Risks
Breaking Hard break for session-id-based lifecycle; needs a dual-support migration window and Claude/Codex/Gemini parity preservation.
Evidence
MCP blog · RC · adoption stats · WorkOS 2026
Rank 2 · Security

MCP03 tool-poisoning & indirect prompt-injection defenses

OWASP's MCP Top 10 lists MCP03 Tool Poisoning (malicious instructions in tool metadata, schemas, or rug-pull updates) as a core risk; a July 2026 CSA note documents live IDE CVEs.

92
Integration shape
A tool-description inspection/sanitization gate before MCP registration, an allowlist mode, runtime anomaly monitoring, and routing sensitive tool calls through Better Agent's approvals surface.
Why now
Better Agent spawns real subprocesses and renders untrusted tool descriptions — "security is the top priority" makes this a direct project mandate.
Risks
Over-aggressive sanitization breaks legitimate tools; needs allowlist + human-in-the-loop tuning; rug-pull detection requires re-validating tool definitions on update.
Evidence
OWASP MCP Top 10 · MCP03 attack page · CSA note (Jul 2026) · NSA→OWASP mapping
Rank 3 · Security / isolation

Sandboxed code execution (E2B / Daytona)

Open-source, purpose-built execution sandboxes — E2B (Firecracker microVMs) and Daytona (self-hostable, ~90ms provisioning) — isolate the commands autonomous agents run.

89
Integration shape
An optional daemons-surface runner that executes untrusted/shell-shaped tool output inside a sandbox instead of the host, wired behind the existing approvals gate.
Why now
Better Agent is explicitly a destructive tool that runs real CLIs; sandboxing is the structural complement to the tool-poisoning gate and confines blast radius on a bad delegation.
Risks
Operational Adds an external dependency and latency; self-hosted Daytona needs infra; must keep secrets/env scrubbing parity and fail-closed on sandbox unavailability.
Evidence
E2B · Daytona · comparison (Northflank)

06 Risks & security cross-cut

Trust-boundary & injection

Better Agent spawns real CLIs and renders untrusted tool/skill metadata. The dominant 2026 threat is indirect prompt injection via tool descriptions, parameter schemas, and rug-pull updates (OWASP MCP03).

  • Gate every MCP/skill registration behind a description-inspection + allowlist pass (Rank 2).
  • Re-validate tool definitions on update to catch rug-pulls.
  • Route sensitive tool calls through the existing approvals surface; fail closed on ambiguity.

Execution isolation

Destructive commands deserve confinement. Open sandboxes (E2B microVMs, Daytona) give a structural blast-radius limit (Rank 3).

  • Run shell-shaped/untrusted tool output in a sandbox, not on the host.
  • Keep env/secret scrubbing parity in the sandbox path — no auth tokens cross the boundary.
  • Fail closed if the sandbox is unavailable; never fall back to host execution silently.

Supply chain

Registry (~9,650 servers) and skill catalogs (hundreds–millions of entries) are now an install surface, not just a discovery one.

  • Prefer verified-namespace servers; mark community items clearly (Ranks 5, 6).
  • Treat SKILL.md bundled scripts as untrusted code; vet before install.
  • Preserve Better Agent's signature/trust-by-source-type model — trust is bound to install path, never self-declared.
